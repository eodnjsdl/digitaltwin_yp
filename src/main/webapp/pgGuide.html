공통 규칙

공통 이름 규칙

  • 문서편집기(Editor) 상에서 사용되는 TAB Key의 기본 Step을 4 Column으로 한다.
  • 줄간 이어쓰기(Warapping Lines)는 아래와 같이 규정한다.
    • 콤파(,) 다음을 나눈다.
    • 연산자 이전을 나눈다
    • 이전 라인과 동일한 레벨의 표현식 시작은 새로운 라인으로 시작한다.
  • 공백 라인(Blank lines)은 논리적으로 관련된 코드섹션을 분리하여 가독성을 높이기 위해 위해 다음과 같이 규정한다.
    • 메소드의 구분
    • 메소드 내의 지역변수 선언부분과 처음 수행 문장 구분
    • Block comment 또는 single-line comment 이전에 구분
    • 메소드 내의 논리적 섹션간 구분
    • Package 선언 후
    • import 선언 후
  • 공백 문자(Blank space)는 다음과 같이 규정한다.
    • 인자 나열 시 콤마(“,”) 다음은 공백 문자를 사용
    • '.'을 제외한 모든 Binary연산자는 Operand 사이에 공백 문자 사용 (단, 증가연산자(“++”), 감소연산자(“--“)등과 같은 단항연산자는 예외)
    • for문 내의 표현식(expression)은 공백문자에 의해 구분
    • 형 변환(casting) 다음에 공백으로 구분
  • 변수 사용과 범위는 다음과 같이 규정한다.
    • 모든 지역변수 선언은 문서(method, function)의 시작부에서 지정하고, 변수의 범위(Variable area) 는 필요한 지역 내로 제한한다.
    • 하나의 단일한 스크립트(Script) 안에 사용되는 변수(Variable)는 지역변수(Local Variable)로 선언(Declaration)하고 사용하기 전에는 반드시 변수(Variable)를 초기화(Initialize)하는 습관을 가져야 한다.

Naming(package, method, function 등)

  • package
    • package naming은 기본으로 egiskorea.com.1차.2차 형태로 구성되며 1,2차 네이밍은 모두 소문자 및 최대 4글자로 제한한다.
    • naming 는 배포한 문서(표준단어 정의서)를 참고한다.
    • 예를들어, 지적이라는 패키지는 표준단어에서 "지적" 으로 검색하여 영문약어의 "LGSR" 를 사용하고, "업소 정보" 의 경우는 "업소"는 BSSH "정보" 는 "INFO" 인데, 이를 줄여서 BSI 로 등록하고 표준용어집에 추가한다.
  • method, attribute
    • java method, attribute 명은 아래의 표를 참고하여 공통 규칙을 통해서 명명한다.
    • 기능명 naming 설명 예시
      조회 select 상세보기 selectExaminationInfo
      목록 select*List 목록 조회 selectExaminationInfoList
      등록 화면 insert*View 등록 화면 호출 insertExaminationInfoView
      등록 insert* 등록 insertExaminationInfo
      수정 화면 update*View 수정 화면 호출 updateExaminationInfoView
      수정 update* 수정 updateExaminationInfo
      다중 삭제 delete*List 목록에서 다중 삭제 deleteExaminationInfoList
      단일 삭제 delete* 상세보기에서 단일 삭제 deleteExaminationInfo
      다중 다운로드 select*ListDownload 목록에서 다중 다운로드 selectExaminationInfoListDownload
      단일 다운로드 select*Download 상세보기에서 단일 다운로드 selectExaminationInfoDownload
      엑셀 업로드 excel*Upload 엑셀데이터 업로드 excelExaminationInfoUpload
  • function(javascript)
    • 패키지내에서 쓰는 공용 함수들은 패키지명.js 로 선언한다.
    • 해당 jsp 파일내에서만 쓰는 스크립트들은 jsp 파일 상단에 선언한다.

웹취약점 표준

크로스사이트스크립트(XSS)

  • jsp 코드내에 EL 문법을 사용하여 값을 출력하지 않고, JSTL 문법을 사용하여 값을 출력한다.
    • 취약한 코드 
      <form name="check" method="post">
<input type="hidden" name="FirstName" id="FirstName" value="${FirstName}"/> # ---> 보안약점
</form>
    • 안전한 코드 
      <form name="check" method="post">
<input type="hidden" name="FirstName" id="FirstName" value="<c:out value="${FirstName}"/>"/> # ---> 수정
</form>

Attribute 표준

상수

  • 상수는 대문자로 사용하며, 여러 단어가 조합되는 경우 각 단어간에 밑줄문자(“_”)를 넣어 표시한다. 
    static final int MIN_WIDTH = 4;
static final int MAX_WIDTH = 999;
static final int GET_THE_CPU = 1;

Method 표준

Query

  • DML(SELECT,UPDATE,DELETE,INSERT)문은 대문자로 작성한다.
  • DML문 내의 테이블명과 칼럼명은 대문자로 작성한다.
  • 다른 스키마의 테이블을 호출할때는 Schema.테이블명으로 작성한다.
  • DML문의 각 단어와 단어의 여백은1칸으로 한다.
    • ',' 는 앞 문자와는 간격을 두지 말고 뒤 문자와는 간격을 1칸으로 한다.
    • 칼럼 List는 되도록 모두 한 줄에 기술한다. (2줄 이상일 경우 첫 줄, 첫 칼럼에 맞춘다)

개발 표준

공통도구

※ 지적/건물 모듈을 샘플로 만들었으니 참조

  • 상단 공통 도구는 egiskorea.com.cmt package에 개발한다.
  • 프로그램가이드 "공통규칙 - Naming - package" 를 참조하여 패키지 및 Controller 파일을 생성하고, method 를 생성한다.
  • 메뉴 button 에 id를 naming 하여 추가하고, data-popup의 value 로 검색하여 이동한 후 width, height 값을 메모한 후 "div class="popup-panel" tag 영역을 잘라내기한다.
  • 패키지 구조에 맞춰 jsp 파일을 생성하여 복사한 내용을 붙여넣기하고, "div class="popup-panel" tag line 부분은 삭제를 한다(/div 도 삭제)
  • /js/map-ui.js 파일내의 function rightPopupOpen 의 switch 문에 메모한 width, height 값을 추가하고, ajax 호출 함수를 추가하고 개발을 진행한다.

1차메뉴(검색, 레이어, 주제도, 업무, 분석)

※ 레이어 모듈을 샘플로 만들었으니 참조

  • 1차메뉴는 동적 영역이 아닌 정적 영역으로 되어있다.
  • Naming 규칙에 맞는 package, java 등을 생성한다.
  • 메뉴 button 의 data-menu value 에 해당하는 class 를 map-ui.js 의 "$("#lnb li[data-menu]").click(function(){" 에 switch 문에 추가하고 ajax 호출 함수를 추가하고 개발을 진행한다.

2차메뉴 bottom popup(left 포함)

※ 업무 > 안전시설물관리 모듈을 샘플로 만들었으니 참조

  • Naming 규칙에 맞는 package, java 등을 생성한다.
  • 공통도구와 동일한 방식으로 진행한다.
  • /js/map-ui.js 파일내의 function bottomPopupOpen 의 switch 문에 메모한 left, width, height 값을 추가하고, ajax 호출 함수를 추가하고 개발을 진행한다.

3차메뉴 left sub popup(right 포함)

※ 국토정보관리 > 행정구역 선택 > 속성정보 더보기

  • Naming 규칙에 맞는 package, java 등을 생성한다.
  • 공통도구와 동일한 방식으로 진행한다.
  • /js/map-ui.js 파일내의 function leftSubPopupOpen 의 switch 문에 메모한 top, left(right), width, height 값을 추가하고, ajax 호출 함수를 추가하고 개발을 진행한다.

개발공유

해당 리스트 화면에 등록 버튼 있을 경우

1. 리스트 화면 표출

  • 초기 전체 리스트 표출.
  • 표출된(전체)리스트에 대한 poi 지도에 표출.
  • 퍼블된 화면에서 체크 박스 및 삭제 버튼 표출않함.

2. 리스트중 한개 목록 클릭시(상세페이지로 이동)

3. 상세페이지 표출

  • 해당위치로 지도이동(화면중심) 및 해당 POI 선택표출.
  • 하단표출 버튼(수정, 삭제, 취소)순으로 표출.
  • 삭제 버튼 클릭시 DB삭제후 목록페이지로 이동.
  • 수정클릭(수정페이지로 이동).

4. 수정페이지 표출

  • 수정페이지에서 하단 버튼(등록, 취소).
  • 취소 버튼 클릭시 상세 페이지로 이동.
  • 등록 버튼 클릭시 DB저장후 목록페이지로.

5. 지도위에 표출된 POI클릭시 속성정보 표출(ex) 시설물 poi클릭시 해당객체에대한 시설물 상세페이지 표출)

해당 리스트 화면에 등록버튼 없을경우

1. 위내용 1~3번까지 동일

  • 삭제기능및 수정 버튼이 없음.
  • 2D/3D 전환시 모든 기능 초기화 ( 활성화되어있는 기능있을시 경고창표출 후 화면 전환 ).
  • 다음주내로 코드 필요한 테이블 코드화 +코드 들어있는테이블 view 테이블 생성 예정.
